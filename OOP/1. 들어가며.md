> LOC : 코드 줄 수
> 
> 왼쪽 : 릴리즈가 높아질 수록 코드 수가 급격하게 증가한다
>
> 오른쪽 : 제품의 기능은 없는데 코드 수는 증가 하고 있다.

## 비용의 증가

## 시간이 흘러 갈수록 코드를 작성하는 시간이 증가하는 이유는?

+ 코드 분석 시간 증가
+ 코드 변경 시간 증가

## 소프트웨어의 가치 : 변화
+ Software maintenance is not "keep it working like before"
+ It is "keep being useful in a changing world"


## 비용과 변화
+ 낮은 비용으로 변화할 수 있어야 함
+ 이를 위한 방법
  - 패러다임
  - 코드, 설계, 아키텍처
  - 업무 프로세스 / 문화

## 객체지향과 비용
+ 캡슐화 + 다형성(추상화)
