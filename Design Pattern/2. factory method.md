# 팩토리 메소드 패턴

+ 어떤 객체, 인스턴스 생성하는 책임을 구체적인 클래스가 아닌 추상적인 인터페이스로 감싸는 방법
  - 구체적인 클래스에 있으면 if else로 계속 코드가 방대해 질것이다.
    - 문제점 : 변경에 닫혀 있지 않게 된다.(개방 폐쇄 원칙 : 확장에는 열려있고 변경에는 닫혀 있어야 한다.)
    - 기존코드에 변경이 없으면서 학장
